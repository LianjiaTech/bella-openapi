# useChatInput 测试指南

## 📍 测试页面访问

启动开发服务器后，访问：
```
http://localhost:3000/zh/playground/test-chat-input
```

## 🧪 测试场景

### 1. 初始状态测试
- [x] status 显示为 "空闲" (idle)
- [x] value 显示为 "(空)"
- [x] canSend 显示为 "✗ false"（红色）
- [x] 发送按钮处于禁用状态（灰色）

### 2. 输入内容测试
**操作步骤：**
1. 在输入框中输入任意文本
2. 观察状态变化

**预期结果：**
- [x] value 显示输入的文本
- [x] canSend 变为 "✓ true"（绿色）
- [x] 发送按钮变为可点击（蓝色）

### 3. 发送流程测试
**操作步骤：**
1. 点击"填充测试文本"按钮
2. 点击"发送"按钮
3. 观察状态变化和日志输出

**预期结果：**
- [x] status 依次变化：idle → 连接中(黄色) → 流式输出中(绿色) → 空闲(灰色)
- [x] 日志显示状态变化时间戳
- [x] 浏览器控制台输出 chunk 数据：`Received chunk: 这是`, `Received chunk: 一段`, 等
- [x] 流式输出过程约持续 2-3 秒（500ms 连接 + 7个chunk×300ms）
- [x] 流式输出期间显示"停止"按钮（红色）

### 4. 中途停止测试
**操作步骤：**
1. 点击"发送"按钮启动流式输出
2. 在 streaming 状态时立即点击"停止"按钮
3. 观察状态和日志

**预期结果：**
- [x] status 立即变为 "已终止"(橙色)
- [x] 日志显示 "[时间] 用户点击停止"
- [x] 浏览器控制台停止输出 chunk
- [x] "停止"按钮消失

### 5. 重新发送测试（streaming 时发送）
**操作步骤：**
1. 点击"发送"按钮启动流式输出
2. 在 streaming 状态时（绿色），再次点击"发送"按钮
3. 观察状态流转

**预期结果：**
- [x] 日志先显示 abort 状态
- [x] 稍后重新开始流式输出（connecting → streaming → idle）
- [x] 浏览器控制台显示新一轮的 chunk 输出

### 6. 清空内容测试
**操作步骤：**
1. 输入内容后，清空输入框
2. 观察状态变化

**预期结果：**
- [x] value 变为 "(空)"
- [x] canSend 变为 "✗ false"
- [x] 发送按钮禁用（灰色）

### 7. 中文输入测试（IME）
**注意：** 此测试需要使用实际输入法（如搜狗拼音、系统中文输入法）

**操作步骤：**
1. 切换到中文输入法
2. 在输入框输入拼音但不按空格（保持拼音未确认状态）
3. 观察 canSend 状态

**预期结果：**
- [x] 输入拼音时 canSend 应为 false（红色）
- [x] 按空格确认中文后 canSend 变为 true（绿色）

**注意：** 当前实现可能需要额外绑定 IME 事件到输入框

### 8. 错误处理测试
**模拟方法：** 需要修改 mockSSEStream 代码临时抛出错误

```typescript
// 在 mockSSEStream 中添加：
if (mockChunks[2]) {
  throw new Error('模拟网络错误');
}
```

**预期结果：**
- [x] status 变为 "错误"(红色)
- [x] 浏览器控制台显示错误信息

## 🔍 观察要点

### 日志面板
- 每次 status 变化都会记录时间戳
- 发送和停止操作都会记录日志
- 可以通过"清空日志"按钮清除历史记录

### 浏览器控制台
- 打开 DevTools Console (F12)
- 观察 `Received chunk:` 日志输出
- 观察错误信息（如果触发）

### React DevTools
- 可以实时观察 hook 内部状态
- 查看 `value`, `status`, `canSend` 的实时值
- 验证 AbortController 的创建和清理

## 📝 测试检查清单

- [ ] 所有初始状态正确
- [ ] 输入/清空内容时 canSend 正确切换
- [ ] 发送流程状态流转正确（idle → connecting → streaming → idle）
- [ ] 中途停止功能正常（abort 状态）
- [ ] streaming 时重新发送先 abort 再重启
- [ ] 日志记录完整准确
- [ ] 浏览器控制台输出 chunk 数据
- [ ] 中文输入法状态正确（需要手动测试）
- [ ] 错误处理正确（需要模拟错误）

## 🐛 已知限制

1. **中文输入测试**：当前测试组件的 `<input>` 元素需要手动绑定 IME 事件才能完整测试 `isComposing` 功能
2. **错误测试**：需要手动修改代码模拟错误场景
3. **网络延迟**：使用固定延迟（500ms + 300ms×7），真实网络会有波动

## 🚀 后续优化建议

1. 在测试组件中添加"模拟错误"按钮
2. 添加延迟时间调节滑块
3. 添加自定义 mock chunk 内容输入
4. 绑定 IME 事件到输入框，完善中文输入测试
5. 添加自动化测试脚本（Vitest + Testing Library）
