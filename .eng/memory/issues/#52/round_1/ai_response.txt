+# Bella OpenAPI
+
+## Multi-Model Support
+
+This API supports specifying multiple models in a single request. This allows the system to try different models if the first choice is unavailable.
+
+### How to Use
+
+When making a completion request, you can specify multiple models in the `model` field using comma-separated values:
+
+```json
+{
+  "model": "gpt-4,gpt-3.5-turbo,claude-2",
+  "messages": [
+    {"role": "system", "content": "You are a helpful assistant."},
+    {"role": "user", "content": "Hello!"}
+  ]
+}
+```
+
+The system will try to use the models in the order specified. If the first model is unavailable, it will try the second, and so on.
+
+### Configuration
+
+The maximum number of models that can be specified in a single request is configurable via the application.yml file:
+
+```yaml
+bella:
+  openapi:
+    max-models-per-request: 3
+```
+
+The default maximum is 3 models per request.
+```
