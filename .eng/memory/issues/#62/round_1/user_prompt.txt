# 项目文件描述


- api/server/sql/02-meta.sql: SQL脚本文件，创建和管理应用中的数据库表结构，涉及模型、能力点和通道等表，定义了相关的字段和约束，用于系统的数据存储及关系定义。

- api/server/sql/04-init.sql: 该文件用于初始化数据库的SQL脚本，插入endpoint、category、model、model_endpoint_rel以及apikey_role等信息，涉及的关键字段包括模型名称、API路径和访问权限等，与数据库交互密切。

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/completion/CompletionAdaptor.java: 无描述

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/completion/OpenAIAdaptor.java: 无描述

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/completion/CompletionRequest.java: 这个文件定义了CompletionRequest类，表示完成请求的数据结构，包含模型ID、用户标识、消息列表等字段。

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/completion/CompletionResponse.java: 该文件定义了CompletionResponse类，包含OpenAPI响应的细节信息与Choice内部类。

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/completion/CompletionProperty.java: 定义Completion配置属性类，实现协议属性接口，包含编码类型、队列名等配置。

- api/server/src/main/java/com/ke/bella/openapi/endpoints/ChatController.java: 这个文件实现了聊天控制器，处理聊天相关的API请求。主要包含completion方法，用于处理请求，并与ChannelRouter、AdaptorManager等交互。

- api/server/src/main/java/com/ke/bella/openapi/protocol/AdaptorManager.java: 定义了AdaptorManager类，用于管理不同协议适配器的注册和获取操作。通过endpoint和protocol识别适配器，并提供相应的服务支持。

- api/server/src/main/java/com/ke/bella/openapi/protocol/IProtocolAdaptor.java: 定义协议适配器接口IProtocolAdaptor，主要提供终端信息和授权请求生成方法，支持不同类型的授权请求构建。

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/IModelFeatures.java: 定义模型功能接口IModelFeatures，包括EndpointModelFeatureType枚举，用于关联端点和模型功能类型。

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/IModelProperties.java: IModelProperties.java定义了模型属性接口及其子类型的枚举类，描述了不同端点对应的模型属性类型，实现IDescription接口，用于在不同的OpenAPI模型间选择适用的属性类型。

- api/sdk/src/main/java/com/ke/bella/openapi/protocol/IPriceInfo.java: 定义价格信息接口IPriceInfo，提供获取单位的方法，并定义相关枚举类型。

- api/server/src/main/java/com/ke/bella/openapi/protocol/log/EndpointLogHandler.java: 定义了EndpointLogHandler接口，以处理特定endpoint的日志，包含日志处理方法。与EndpointProcessData类交互。

- api/server/src/main/java/com/ke/bella/openapi/protocol/metrics/MetricsResolver.java: 定义MetricsResolver接口，包含度量数据的处理方法。用于解析与EndpointProcessData相关的度量信息。

- api/sdk/src/main/java/com/ke/bella/openapi/metadata/MetadataFeatures.java: 定义了枚举类MetadataFeatures，列举不同的元数据特性和相关端点。提供了特性列表和验证方法。

- api/sdk/src/main/java/com/ke/bella/openapi/common/EntityConstants.java: 此文件定义了系统中的常量，包括实体类型、角色、优先级等，使用多个枚举类组织常量数据，与系统配置强相关。

- api/server/src/main/java/com/ke/bella/openapi/endpoints/MetadataController.java: 这个文件定义了REST API控制器MetadataController，负责处理元数据相关的HTTP请求，包括能力点、模型、渠道和类目等的CRUD操作。主要类/方法：listEndpointDetails、listModel、createPrivateChannel等。与服务层交互以实现具体功能。

- api/server/src/main/java/com/ke/bella/openapi/console/MetadataConsoleController.java: 此文件定义了MetadataConsoleController类，负责管理元数据，包括端点、模式、频道和类目等的增删改查操作。主要通过RESTful API接口与前端通信，调用各个Service类进行业务处理。

- web/src/app/meta/page.tsx: 该文件是一个React组件，用于显示元信息页面，包括侧边栏和主显示区域，使用Hooks管理状态，获取并展示分类树及端点信息。与多个组件配合展示数据。

- web/src/app/playground/page.tsx: 这个文件定义了Playground组件页面，用于展示模型选择和能力点的互动。通过组件如ClientHeader和Sidebar进行界面组织并利用API来获取数据。

- web/src/components/meta/endpoint-details.tsx: 展示和过滤API端点的详细信息及关联模型的组件，支持用户自定义筛选条件。

- web/src/components/playground/ChatCompletionsProcessor.ts: ChatCompletionsProcessor类用于发送和处理聊天生成的流式响应，管理连接的状态变化，解析响应数据。

- api/pom.xml: api/pom.xml 是 Maven 项目的配置文件，定义了项目的模块、依赖管理和构建插件。主要功能是管理项目的依赖和构建过程，包含的关键配置有 Spring Boot、Spring Cloud 以及各种插件如 maven-source-plugin 和 maven-gpg-plugin，和 server、sdk、spi 模块的交互显著。

- api/server/pom.xml: 此文件是server模块的Maven POM文件，继承自父POM(bella-openapi)，定义了模块的依赖，如Spring Boot、Jooq等，并配置了数据库连接属性以及代码生成插件。

64     public Object completion(@RequestBody CompletionRequest request) {
65         String endpoint = EndpointContext.getRequest().getRequestURI();
66         String model = request.getModel();
67         
68         // Handle multi-model requests
69         if (model != null && model.contains(",")) {
70             
71             // Try each model in order
72             String[] models = model.split(",");
73             int maxNum = models.length;
74             // Validate model count
75             if (models.length > maxModelsPerRequest) {
76                 LOGGER.warn("请求模型数量超过最大限制: " + maxModelsPerRequest);
77                 maxNum = maxModelsPerRequest;
78             }
79             Exception lastException = null;
80             
81             for (int i = 0; i < maxNum; i++) {
82                 String singleModel = models[i];
83                 String trimmedModel = singleModel.trim();
84                 // Create a copy of the request with just this model
85                 request.setModel(trimmedModel);
86                 
87                 try {
88                     // Try to process with this model
89                     return processCompletionRequest(endpoint, trimmedModel, request);
90                 } catch (Exception e) {
91                     // Log the exception and continue to the next model
92                     lastException = e;
93                 }
94             }
95             
96             // If we get here, all models failed
97             if (lastException != null) {
98                 throw ChannelException.fromException(lastException);
99             } else {
100                 throw new BizParamCheckException("所有指定的模型都无法处理请求");
101             }
102         }
103         
104         // Single model processing
105         return processCompletionRequest(endpoint, model, request);
106     }

112         EndpointContext.setEndpointData(channel);
113         if(!EndpointContext.getProcessData().isPrivate()) {

115         }
116         Object requestRiskData = safetyCheckService.safetyCheck(SafetyCheckRequest.Chat.convertFrom(request,

121         String url = processData.getForwardUrl();

212     // 创建取消控制器
213     this.controller = new AbortController();
214     const signal = this.controller.signal;
215 
216     // 准备请求参数
217     const requestParams: RequestInit = {
218       method: 'POST',
219       headers: {
220         'Content-Type': 'application/json',
221         ...this.config.headers
222       },
223       body: JSON.stringify(request),
224       credentials: 'include', // 添加此行确保携带cookie
225       signal
226     };
227 
228     // 设置超时
229     const timeoutId = setTimeout(() => {
230       this.cancel('请求超时');
231     }, this.config.timeoutMs);
232 
233     try {
234 
235       // 发送请求并获取流式响应
236       const response = await fetch(this.config.url, requestParams);
237 
238       clearTimeout(timeoutId);
239 
240       if (!response.ok) {
241         const errorText = await response.text();
242         throw new Error(`请求失败: ${response.status} ${response.statusText} - ${errorText}`);
243       }
244 
245       if (response.body === null) {
246         throw new Error('响应体为空');
247       }
248 
249       // 触发开始事件
250       this.setState(ChatCompletionsState.CONNECTED);
251 
252       // 开始处理流式响应
253       await this.processStream(response.body);
254 
255     } catch (error) {
256       clearTimeout(timeoutId);
257       if (signal.aborted) {
258         // 请求已被取消，不需要处理错误
259         return;
260       }
261 
262       this.setState(ChatCompletionsState.ERROR);
263       this.emit(ChatCompletionsEventType.ERROR, error instanceof Error ? error.message : String(error));
264       // 在错误处理后自动重置状态到IDLE，这样可以继续发送新的请求
265       setTimeout(() => {
266         if (this.state === ChatCompletionsState.ERROR) {
267           this.setState(ChatCompletionsState.IDLE);
268         }
269       }, 500);
270     }
271   }

273   /**
274    * 处理流式响应
275    * @param body 响应体
276    */
277   private async processStream(body: ReadableStream<Uint8Array>): Promise<void> {
278 
279     // 设置状态
280     this.setState(ChatCompletionsState.RESPONDING);
281     this.emit(ChatCompletionsEventType.START, {});
282 
283     const reader = body.getReader();
284     const decoder = new TextDecoder();
285 
286     try {
287       // 创建SSE解析器
288       const parser = createParser({
289         onEvent: (event) => {
290           // 所有的onEvent调用都是事件
291           {
292           const data = event.data;
293 
294           // 检查是否为结束信号
295           if (ChatCompletionsResponseParser.isDone(data)) {
296             this.setState(ChatCompletionsState.FINISHED);
297             this.emit(ChatCompletionsEventType.FINISH, {});
298             return;
299           }
300 
301           try {
302             // 解析JSON数据
303             const response = ChatCompletionsResponseParser.parseData(data);
304 
305             // 处理数据
306             if (response && response.choices.length > 0) {
307               const choice = response.choices[0];
308 
309                 // 处理正常内容
310                 if (choice.delta?.content !== undefined) {
311                   // 累积响应文本
312                   this.responseText += choice.delta.content;
313 
314                   // 触发内容增量事件
315                   this.emit(ChatCompletionsEventType.DELTA, {
316                     content: choice.delta.content,
317                     role: choice.delta.role || 'assistant',
318                     response: response
319                   });
320                 }
321 
322                 // 处理深度思考内容
323                 if (choice.delta?.reasoning_content !== undefined) {
324                   // 触发深度思考内容增量事件
325                   this.emit(ChatCompletionsEventType.DELTA, {
326                     reasoning_content: choice.delta.reasoning_content,
327                     role: choice.delta.role || 'assistant',
328                     isReasoningContent: true,
329                     response: response
330                   });
331                 }
332 
333                 // 处理完成原因
334                 if (choice.finish_reason) {
335                   this.setState(ChatCompletionsState.FINISHED);
336                   this.emit(ChatCompletionsEventType.FINISH, { reason: choice.finish_reason });
337                 }
338             }
339           } catch (error) {
340             return;
341           }
342           }
343         },
344         onError: (err) => {
345           this.setState(ChatCompletionsState.ERROR);
346           this.emit(ChatCompletionsEventType.ERROR, err instanceof Error ? err.message : String(err));
347         }
348       });
349 
350       // 流式处理返回数据
351       while (true) {
352         const { done, value } = await reader.read();
353         if (done) {
354           break;
355         }
356 
357         // 将二进制数据转换为文本并传递给解析器
358         const text = decoder.decode(value, { stream: true });
359 
360         parser.feed(text);
361       }
362 
363       // 确保最终状态为完成
364       if (this.state !== ChatCompletionsState.FINISHED) {
365         this.setState(ChatCompletionsState.FINISHED);
366         this.emit(ChatCompletionsEventType.FINISH, {});
367       }
368 
369     } catch (error) {
370       if (this.controller && !this.controller.signal.aborted) {
371         this.setState(ChatCompletionsState.ERROR);
372         this.emit(ChatCompletionsEventType.ERROR, error instanceof Error ? error.message : String(error));
373 
374         // 在流处理错误后也自动重置状态
375         setTimeout(() => {
376           if (this.state === ChatCompletionsState.ERROR) {
377             this.setState(ChatCompletionsState.IDLE);
378           }
379         }, 500);
380       }
381     } finally {
382       reader.releaseLock();
383     }
384   }
385 
386   /**
387    * 取消请求
388    * @param reason 取消原因
389    */
390   public cancel(reason?: string): void {
391     if (this.controller) {
392       this.controller.abort();
393       this.controller = null;
394       this.setState(ChatCompletionsState.CANCELLED);
395       // 用户取消不显示为错误，而是完成状态
396       this.emit(ChatCompletionsEventType.FINISH, { cancelled: true, reason: reason || '用户取消' });
397     }
398   }
